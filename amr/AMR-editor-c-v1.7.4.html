Help

  • What is this?   An editor to build abstract meaning representations (AMRs) written by Ulf at ISI.
  • First time user?
    1. This simple example shows how to build an AMR for "The boy wants to see the girl."
    2. This next example includes a shortcut for adding named entities (NEs) "John F. Kennedy was murdered."
      as well as a replace (= change) operation.
    3. Check out the AMR Guidelines, the AMR Dictionary, YouTube videos about AMR annotation, the AMR Editor Manual, the QuickRef (popup), the FAQs and the Release Notes.
  • Why are there two input methods?
    1. The text command input method (see top input line) allows Unix style commands, suited for fast entry of common commands and by the experienced user. One way to learn how to use text command input method is to first use the easier template-based method, and after submitting, to study the automatically generated "Last command" line.
    2. The template-based command input methods are activated by clicking on one of the blue action template boxes such as "top" or "add". They provide more guidance, often through mouse-overs, and will be easier for new users. When replacing (= changing) concepts, strings, numbers and roles in the AMR, the user can easily identify the element to be changed by clicking on the item to be changed.
